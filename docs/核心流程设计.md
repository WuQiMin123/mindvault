### 步骤摘要

1. **去重检测**: 查 URL + content_hash，已有直接返回
2. **入库**: 创建 Link(pending)，返回 202
3. **调度**: TaskRunner 启动异步任务
4. **抓取**: Crawler 按 URL 路由到对应实现，获取标题+正文，清洗 HTML
5. **判空**: 有正文 → 继续 LLM 处理；正文为空 → status=no_content，自动关联"未成功"标签，跳过 LLM
6. **摘要**: LLM 生成摘要
7. **打标签**: LLM 建议 1-3 个标签，存入 `tags_ai` 字段（待用户确认）
8. **完成**: 更新 status = completed

### 异常处理

| 异常 | 处理方式 | 用户可见 |
|------|----------|----------|
| 抓取失败 | status = failed, 记录 error | "无法访问该链接" |
| LLM 限流 | 指数退避重试 2 次 (1s → 3s) | 内部透明 |
| LLM 调用失败 | 2 次重试后仍失败 → status = failed | "AI 处理失败，稍后重试" |
| 抓取内容为空 | status = no_content（保留链接，跳过 LLM） | 仅标题/链接，无正文 |
| 内容清洗异常 | 跳过该条，继续处理 | 正常完成但无摘要 |

## 流程 2: 快速暂存链接

先存后读：看到有意思的链接但没时间看，一键暂存到"待处理"标签，有空再触发分析。

```
用户复制链接             前端                后端 API
 │                       │                   │
 │ 粘贴链接               │                   │
 │ 点击"暂存"            │                   │
 │──────────────────────>│                   │
 │                       │ POST /ingest/bookmark
 │                       │──────────────────>│
 │                       │                   │ 创建 Link
 │                       │                   │  ├─ url, title(从页面抓标题)
 │                       │                   │  ├─ content = null
 │                       │                   │  ├─ status = completed
 │                       │                   │  └─ 自动关联"待处理"标签
 │                       │                   │
 │                       │ 201 { id }        │
 │                       │<──────────────────│
 │ 显示"已暂存"          │                   │
 │<──────────────────────│                   │
 │                       │                   │
 │ ~~~ 之后某天 ~~~      │                   │
 │                       │                   │
 │ 点击"开始分析"        │                   │
 │──────────────────────>│                   │
 │                       │ POST /links/{id}/analyze
 │                       │──────────────────>│
 │                       │                   │ status = pending
 │                       │                   │ 触发异步管线 (同流程 1)
 │                       │ 202 { status }    │
 │                       │<──────────────────│
 │ 显示"分析中"          │                   │
 │<──────────────────────│                   │
```

### 暂存清单

用户可在"待处理"标签页看到所有暂存的链接，每条有"开始分析"按钮。
分析完成后，自动移除"待处理"标签，归入 AI 建议的标签。

手动添加支持三种场景：

| 场景 | 说明 | API |
|------|------|-----|
| **粘贴纯文本** | 从别处复制的文章/笔记，直接粘贴 | POST /ingest/text |
| **写笔记** | 自己写的内容，无外部链接 | POST /ingest/text |
| **补录链接** | 链接提取失败后，用户手动填入正文 | PATCH /links/{id} (更新 content + 重走 LLM) |

手动添加流程：

```
用户粘贴/输入标题+正文
    │
    ▼
POST /api/v1/ingest/text
    │
    ▼
创建 Link(status = completed)
 ├── content = 用户输入（全文保存）
 ├── summary = 空（待 LLM 生成）
 ├── source = "manual"
 ├── url = 空（纯文本/笔记场景）
    │
    ▼
后台异步执行 LLM 处理
 ├── 生成摘要 → 写入 summary
 └── 建议标签 → 写入 tags_ai
    │
    ▼
用户在前端看到逐步填充的摘要和标签
```

## 流程 3: 标签内聚合总结

```
用户在标签页
    │
    ▼

    │
    ▼

    ▼

[摘要2]
... "
    │
    ▼
返回聚合总结文本
```

## 流程 4: 去重检测

```
提交链接 / 内容
    │
    ▼
两层去重:
    1. URL 去重: SELECT * FROM links WHERE url = ?
    2. 内容去重: MD5(content) → 查 content_hash
    │
    ├── 已存在 → 返回 { exists: true, link_id: 已有记录的 id }
    └── 不存在 → 正常入库
```

## 流程 5: 服务重启恢复

```
FastAPI 启动 → lifespan 触发
    │
    ▼
TaskRunner.start() 执行:
    │
    SELECT * FROM links WHERE status IN ('pending', 'processing')
    │
    ├── 有未完成任务
    │   → 对每条记录提交 asyncio.create_task(_run_pipeline(id))
    │   → 日志: "恢复了 N 条未完成任务"
    │
    └── 无未完成任务
        → 日志: "没有需要恢复的任务"
```

### 边界情况

| 场景 | 行为 |
|------|------|
| 正常完成 | status = completed, 不触发恢复 |
| 处理到一半重启 | status = processing, 自动恢复 |

## 流程 6: 内容安全清洗

```
Crawler 获取原始 HTML
    │
    ▼
sanitize_content(html):
    1. 移除 <script>...</script> 以及内容
    2. 移除 <style>...</style> 以及内容
    3. 移除所有其他 HTML 标签
    4. 解码 HTML 实体 (&amp; → &)
    5. 合并空白字符
    │
    ▼
存入 content（纯文本，不含任何标签）
    │
    ▼
前端展示: React 默认使用 textContent，自动转义
```

## v2 规划（MVP 后）

以下功能已明确范围但不在 MVP 中：

| 功能 | 范围 | 搁置原因 |
|------|------|----------|
| 浏览器插件 | Chrome Manifest V3 | 开发+审核周期长 |
| PDF 上传 | PyMuPDF 文字提取 | 图片型 PDF 无法处理，效果参差 |
| 数据导出 | JSON / Markdown | 不影响核心价值验证 |
| 阅读队列 | 已读/未读标记 | 体验优化，非核心 |
| 批量提交 | 一次提交多个链接 | 可以用多次操作替代 |
