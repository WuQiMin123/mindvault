# 核心流程设计

> v0.2.0 — 输入聚焦于浏览器插件提取和手动文本输入。

---

## 流程 1: 手动录入文本

Web 端提交页或插件弹出窗，用户输入标题+正文+标签，直接入库。

```
用户                  前端 / 插件             后端 API
 │                       │                      │
 │ 输入标题+正文+标签      │                      │
 │──────────────────────>│                      │
 │                       │ POST /ingest/text    │
 │                       │─────────────────────>│
 │                       │                      │ 创建 Link
 │                       │                      │  ├─ title, content
 │                       │                      │  ├─ source = "manual"
 │                       │                      │  └─ status = completed
 │                       │                      │ 关联指定标签
 │                       │                      │
 │                       │ 201 { id, status }   │
 │                       │<─────────────────────│
 │                       │                      │
 │ 显示"已保存"           │                      │
 │<──────────────────────│                      │
```

### 步骤摘要

1. **输入**: 用户在 Web 端或插件输入标题、正文，从已有标签选择或创建新标签
2. **校验**: 标题和正文必填，标签可选
3. **入库**: 创建 Link(status=completed)，关联标签
4. **返回**: 返回 ID 和状态

### 异常处理

| 异常 | 处理方式 | 用户可见 |
|------|----------|----------|
| 内容为空 | 422 参数校验失败 | "内容不能为空" |
| 标题为空 | 422 参数校验失败 | "标题不能为空" |

## 流程 2: 浏览器插件提取内容

插件提供两种输入方式：右键菜单保存选中文本、B站视频页自动提取字幕。

```
用户在网页操作              插件                      后端 API
 │                           │                           │
 │ 选中文本 → 右键保存        │                           │
 │──────────────────────────>│                           │
 │                           │ 打开 popup，预填内容        │
 │                           │ 加载已有标签列表            │
 │                           │                           │
 │ 选择/创建标签 → 点保存     │                           │
 │──────────────────────────>│                           │
 │                           │ POST /ingest/text         │
 │                           │──────────────────────────>│
 │                           │ 201                       │
 │                           │<──────────────────────────│
 │                           │                           │
 │ 显示"保存成功"             │                           │
 │<──────────────────────────│                           │
```

### 插件功能清单

| 功能 | 触发方式 | 内容来源 |
|------|----------|----------|
| 选中文本保存 | 右键菜单 / Ctrl+Shift+V | 网页选中文本 |
| B站字幕提取 | 在 B站视频页点击图标 | CC 字幕 API |
| 标签选择 | popup 自动加载 | GET /tags |

## 流程 3: 标签内聚合总结

```
用户在标签页
    │
    ▼
GET /api/v1/tags/{id}/summary
    │
    ▼
查 link_tag → 取该标签下所有 Link
    │
    ▼
把各条 summary 合并 → 发给 LLM

Prompt:
"以下是一组相关内容的摘要，请综合总结主要观点和关联：
[摘要1]
[摘要2]
... "
    │
    ▼
返回聚合总结文本
```

## 流程 4: 内容管理

支持列表浏览、详情查看、编辑、删除、标签管理。

### 列表浏览
- 分页展示，每页 20 条
- 支持按标签、来源、状态筛选
- 关键词搜索

### 内容详情与编辑
- 查看全文、摘要、标签
- 添加/移除标签（可选已有标签或当场创建新标签）
- 修改内容正文和标题
- 删除内容（确认后物理删除）

### 标签管理
- 列表展示所有标签（含内容数量）
- 创建、编辑名称/颜色、删除
- 点击标签查看该标签下所有内容

## v0.2.0 vs v0.1.0 变化

以下功能已在 v0.2.0 中移除：

| 功能 | 移除原因 |
|------|----------|
| 链接提交（ingest/link） | 反爬成本高，用户手动选择更精准 |
| 快速暂存（ingest/bookmark） | 依赖链接提交管线 |
| 去重检测（ingest/check） | 无链接输入场景不需要 |
| 后台抓取管线（TaskRunner + Crawler） | 不再需要异步抓取处理 |
| 重新分析（links/{id}/analyze） | 无后台处理管线 |
