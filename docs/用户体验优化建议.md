# 用户体验优化建议

## 用户旅程地图

```
发现内容 → 想要保存 → 保存到系统 → 等待处理 → 回顾阅读 → 查找检索 → 复用知识
                                               ↓
                                         长期维护
```

每个环节都有可优化的体验点。

---

## 一、保存环节 — 降低摩擦

### 1.1 浏览器插件 (已规划)
右键 / 快捷键一键保存，**不需要打开 Web 端**。

### 1.2 URL 智能识别
- 用户粘贴链接时自动识别平台，显示来源图标
- 如果粘贴的是纯文本（非 URL），自动切到文字输入模式
- 如果粘贴的是 B站合集/知乎专栏，提示"检测到合集，是否全部导入？"

### 1.3 批量提交
- 一次粘贴多个链接（换行分隔），全部提交
- 拖动文件到页面，自动识别是 PDF 还是图片

### 1.4 提交时即时反馈
- 粘贴链接后立即显示"检测中..."，快速显示标题预览
- 已收藏过的链接直接提示"已存在，打开已有记录"

---

## 二、等待处理 — 减少焦虑

### 2.1 实时进度展示
- 不只是一个 "pending" → "completed" 的状态
- 显示当前步骤：抓取中 → 生成摘要 → 打标签 → 向量化 → 完成
- 抖音等长耗时任务显示估算剩余时间

### 2.2 失败友好提示
- 抓取失败时不只是报错，而是提供替代方案：
  - "页面需要登录才能访问，试试用浏览器插件保存？"
  - "抖音视频下载失败，可能是链接已失效"

### 2.3 完成后通知
- 状态变为 completed 后，页面标题闪动或显示通知
- 新入库的内容在列表中有"NEW"标识

---

## 三、浏览查阅 — 发现与回顾

### 3.1 阅读队列
- **首页"待读"板块**：显示 is_read=false 的内容，优先消耗
- 阅读后标记为已读
- **"随机回顾"按钮**：随机挑一条历史内容，帮助唤醒记忆

### 3.2 多种视图
| 视图 | 适用场景 |
|------|----------|
| 列表视图（默认） | 快速扫描标题和摘要 |
| 卡片视图 | 图文预览 |
| 时间线视图 | 按时间回溯知识积累过程 |

### 3.3 阅读体验
- **阅读模式**：点开内容后进入类似 Safari Reader 的干净阅读界面
- **原文跳转**：点击"查看原文"在新标签页打开原始链接
- **内部划线/批注**（v2 考虑）：选中文字添加笔记

### 3.4 搜索
- **全局搜索快捷键 `Cmd+K`**：任何时候唤起搜索框
- 支持搜索标题、内容、标签
- 搜索结果高亮关键词

---

## 四、标签管理 — 组织与整理

### 4.1 AI 标签用户确认流程
```
AI 建议: [AI, 深度学习, 论文]  □AI □深度学习 □论文  ✓确定
```
- 展示 AI 建议的标签，用户勾选确认再写入
- 用户也可以直接输入自定义标签

### 4.2 标签操作
- **标签合并**：拖拽标签 A 到标签 B → 合并（A 下所有内容归入 B，删除 A）
- **标签重命名**：直接改名
- **标签颜色**：自动分配随机颜色，用户可自定义

### 4.3 标签层级 (v2)
- 支持父子标签：AI > 机器学习 > 深度学习
- 父标签下显示所有子标签的内容聚合

## 五、知识复用 — 让收藏变有用

### 5.1 标签聚合总结
- 点进标签先看到一句话总结："该标签下 15 篇内容主要围绕..."
- LLM 综合分析所有内容后生成

### 5.2 关联推荐
- 查看一篇文章时，底部推荐相关文章
- 基于标签重叠 + 向量相似度

### 5.3 知识图谱 (v2)
- 可视化内容之间的关联：共享标签、相似主题
- 类似小说分析器的关系图，但这里节点是文章，边是标签/相似度

### 5.4 每周回顾邮件 (v2)
- 每周自动总结：这周收藏了什么、哪篇最有价值
- 推送"你可能忘了"的历史内容

---

## 六、数据掌控 — 安全感

### 6.1 导出（已规划）
- 一键导出 JSON 或 Markdown
- 数据完全归你

### 6.2 统计仪表盘
```
今日收藏: 3    总计收藏: 127    标签数: 18
最近活跃标签: AI(15)  后端(8)  论文(12)
阅读完成率: 68%
```
- 看到自己的知识库在成长，有正反馈

### 6.3 回收站
- 删除的内容先进回收站，30 天后自动清理
- 防止误删
