## 关键设计原则

1. **用户主动输入 > 自动抓取** — 反爬成本高，用户手动选中更精准
2. **数据不丢** — SQLite WAL 模式持久化，数据在本地磁盘
3. **插件覆盖主要场景** — 右键保存 + B站字幕，网页内容可直接入库
4. **纯本地部署** — 不依赖外部服务（LLM 除外）

## 分层说明

### 1. 前端层 (Frontend)
- **框架**: Next.js App Router + TypeScript
- **样式**: Tailwind CSS + shadcn/ui
- **页面路由**:
  - `/` — 仪表盘（统计 + 最近内容）
  - `/ingest` — 提交文本（标签选择 + 新建）
  - `/tags` — 标签列表
  - `/tags/[id]` — 标签详情 + 聚合总结
  - `/links/[id]` — 内容详情（编辑、标签管理）
  - `/search` — 搜索

### 2. API 路由层 (Routers)
- 职责：参数校验、请求/响应序列化
- 不包含业务逻辑
- 统一错误格式

### 3. 数据层 (Data)
- **SQLite + SQLAlchemy (async)**: 结构化数据存储
- **WAL 模式**: 允许并发读 + 写
- 数据模型: Link —< LinkTag >— Tag

## 数据模型

```
Link ──< LinkTag >── Tag
```

| 模型 | 说明 |
|------|------|
| Link | 内容条目（标题、正文、来源、摘要、状态） |
| Tag | 标签（名称、颜色），全局唯一 |
| LinkTag | 多对多关联表 |

## 输入方式

| 方式 | 入口 | source 值 |
|------|------|-----------|
| Web 端手动输入 | `/ingest` 页面 | `manual` |
| 插件选中文本 | 右键菜单 / Ctrl+Shift+V | `extension` |
| 插件 B站字幕 | B站视频页图标 | `bilibili` |
