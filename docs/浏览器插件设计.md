# 浏览器插件设计

## 定位

选中网页上的任意文本 → 一键保存到 MindVault。

解决后端抓取遇到反爬的问题，同时也比全页提取更精准——用户只存自己关心的内容。

## 用法

```
1. 在网页上选中一段文字
2. 右键菜单 → "保存到 MindVault"（或快捷键 Ctrl+Shift+V）
3. 弹出浮窗，自动带入选中的文本 + 页面标题 + 页面 URL
4. 用户可选添加标签
5. 点保存 → 调 API 保存 → 跳转到 Web 端查看
```

## 触发方式

- **右键菜单**：选中文本后右键 → "保存到 MindVault"
- **快捷键**：选中文本后按 Ctrl+Shift+V
- **点击图标**：没选中文本时点图标，提示"请先选中内容"

## 流程

```
用户选中文本 → 右键/快捷键
       │
       ▼
background service worker 接收命令
    （右键菜单直接带 selectionText，快捷键注入脚本取 text）
       │
       ▼
存入 chrome.storage.session，打开 popup
       │
       ▼
popup.html 读取数据并展示: { text, title, url }
       │
       ▼
popup 显示内容（可编辑文本框）+ 页面标题 + URL + 标签选择器
       │
       ▼
用户可修改内容、添加标签
       │
       ▼
点"保存"
       │
       ▼
调 API POST /ingest/text
       │
       ▼
成功 → 打开 Web 端查看
```

## 文件结构

```
browser-plugin/
├── manifest.json          # Manifest V3
├── popup.html             # 弹出浮窗
├── popup.js               # 浮窗逻辑：显示选中内容 + 标签
├── background.js          # 右键菜单注册 + 快捷键监听
├── options.html           # 配置页
├── options.js             # 配置页逻辑
└── icons/
    ├── icon16.png
    ├── icon48.png
    └── icon128.png
```

## API 接口

使用现有接口：

```
POST /api/v1/ingest/text
{
  "title": "页面标题",
  "content": "选中的文本",
  "url": "当前页面 URL",
  "tags": ["用户选的标签"]
}
```

## 配置

API 地址配置在插件设置页，存入 chrome.storage.sync。

后续 Phase 4（多用户）可加入 Token 认证。
